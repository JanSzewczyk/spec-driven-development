## [0.1.0] — 2026-06-04

### Added

- Initial release of the SDD Framework for Claude Code.
- `bootstrap.sh` — one-command installer (local or remote via `curl`).
- `template/` — the framework payload copied into target projects:
  - `CLAUDE.md.template` — project constitution scaffold (<2,500 tokens).
  - `specs/_template.md` — feature specification template.
  - `.claude/skills/sdd-doctor/` — the SDD doctor skill with 10-point readiness check and `init`/`fix` modes (auto-detect installed plugins + stack).
  - `.claude/commands/` — 8 slash commands: `sdd-doctor`, `constitution`, `spec`, `clarify`, `plan`, `tasks`, `implement`, `review`, `analyze`.
  - `.claude/agents/` — 4 generic verification sub-agents:
    - `sdd-spec-guard` — diff vs Acceptance Criteria.
    - `sdd-drift-detector` — spec ↔ plan ↔ tasks ↔ code consistency.
    - `sdd-reviewer` — final quality audit (verdict GO/NO-GO).
    - `sdd-ui-critic` — visual review via browser MCP + Storybook (opt-in, skips on missing MCP).
  - `.claude/hooks/` — `typecheck.py`, `lint.sh` (PostToolUse, exit 2 blocks Claude).
  - `.claude/settings.json` — PostToolUse hook wiring.
  - `.claude/capabilities.md.template` — hybrid capability registry (auto-generated + `<!-- user-override -->` sections).
- Contract-first TDD for UI components — `/tasks` emits a 3-task chain (`ui-contract` → `ui-component-test` → `ui-component`) so React/Next.js components get a meaningful red phase. Props interface lives inline in `.tsx` (no separate `.types.ts`).
- Three-tier agent hierarchy: Orchestrator → Specialist agents (plugin-based) → Verification agents (SDD-owned), with Skills loaded on demand.
- Per-task routing via YAML tags (`type`, `agent`, `skills`) in `tasks.md` — automatically auto-generated by `/tasks` from `capabilities.md` rules.
- Conventional Commits-style branches (`feat/...`, `fix/...`) enforced by `/spec`.
- Comprehensive README with quick start, end-to-end example, slash command reference, sub-agent reference, best practices, anti-patterns, design decisions, and troubleshooting.
